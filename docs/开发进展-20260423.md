## 3. 产物参数详情（便于后续扩展）

后端 `PRODUCT_PARAMS` 字典（`operators.py`）与前端 `PRODUCT_PARAM_DEFS`（`ImageProcessingModule.tsx`）**必须一一对应**，字段名与默认值要严格对齐，否则"全部默认值时不请求后端"的判断会失效。

| Key | 字段 | 默认值 | 范围 | 含义 |
|---|---|---|---|---|
| `original` | brightness | 0 | -50~50 | 亮度偏移 |
|  | contrast | 1.0 | 0.5~2.0 | 对比度倍率（以 128 为中心） |
|  | saturation | 1.0 | 0~2.0 | HSV S 通道倍率 |
|  | gamma | 1.0 | 0.3~3.0 | 伽马校正 |
| `sharpen` | amount | 1.45 | 0.3~3.0 | Unsharp 强度 |
|  | radius | 1.9 | 0.5~5.0 | 模糊半径 |
| `microtrace` | clip_limit | 3.3 | 0.5~8.0 | CLAHE 对比 |
|  | tile_size | 10 | 2~24 | CLAHE 分块 |
|  | bg_sigma | 60 | 10~120 | 背景均匀化核 |
|  | gain | 140 | 60~220 | 归一化增益 |
|  | blend_alpha | 0.35 | 0~1 | 与原图混合（0=纯处理） |
| `grayscale` | clip_limit | 2.0 | 0.5~6.0 | CLAHE 对比 |
|  | tile_size | 8 | 2~24 | CLAHE 分块 |
|  | sharpen_amount | 0 | 0~0.6 | 纹理锐化 |
| `rubbing` | bg_sigma | 55 | 10~120 | 背景均匀化核 |
|  | gain | 133 | 60~220 | 归一化增益 |
|  | clip_limit | 2.5 | 0.5~6.0 | CLAHE 对比 |
|  | tile_size | 12 | 2~24 | CLAHE 分块 |
|  | sharpen_amount | 0.1 | 0~0.5 | 拓片锐化 |

## 4. UI 设计约定（后续模块都要对齐）

图像处理模块的布局已固化为"模板"，新模块应保持一致：

- 外层 `<div className="flex h-screen flex-1">`
- 左侧 `<aside className="flex h-screen w-72 shrink-0 flex-col overflow-hidden border-r border-paper-300/70 bg-paper-50/60">`
  - 顶部文物下拉：`<RelicPicker className="w-full" />`
  - 功能按钮/分组：外层 `mx-3 rounded-xl bg-paper-100/80 ring-1 ring-paper-300/60`
  - 激活态：`bg-ochre-400/15 text-ochre-700 ring-1 ring-ochre-500/30`
  - 次要文字：`text-[10px] text-ink-400`
  - **不允许出现滚动条**；若内容超出可用 `[scrollbar-width:none] [::-webkit-scrollbar]:hidden` 隐藏
- 右侧 `<section className="flex min-h-0 min-w-0 flex-1 flex-col">` + 内部图像视口
- 不再有顶部横向 header / 不再显示"后端在线"chip / 不再有底部文件名 · 日期 · 下载

配色规范：
- 主色 `ochre`（琥珀）用于激活/高亮
- `bamboo`（竹绿）用于"对比"等辅助功能态
- `seal`（印朱）用于错误/警示
- `ink`（墨黑）分级用于正文与次要文本
- `paper`（米黄）用于容器背景与边框

## 6. 下一步规划：字迹标注模块重写

### 6.1 需求定位（已与用户对齐）

碑刻/画像石研究场景下的三类刚需：
1. **字形释读留档**：在图上框出字 → 记录释读 → 可复核
2. **疑难点讨论**：挂在具体字上的存疑/异体/工艺备注
3. **导出成果**：标注数据可导出 JSON/CSV 生成释读报告

### 6.2 技术方案

**去掉 OpenSeadragon**，改用与多图层模块相同的轻量方案（参考已删除的 `MultiLayerModule.tsx` 中的 `LayerViewport` 实现）：
- `<img>` 显示底图（原始高清）
- 容器级 `transform` / 绝对定位 + `scale` 实现缩放平移
- 标注框用 CSS `%` 定位（基于归一化 bbox 坐标）跟着图一起缩放
- `ResizeObserver` 自动 fit 到容器

### 6.3 布局（与图像处理一致）

```
┌─────┬──────────────────┬─────────────────────┐
│模块 │   左侧栏 w-72     │   右侧图像视口       │
│Nav  │                  │                     │
│ 20px│ 文物下拉          │ ┌─────────────────┐ │
│     │ ─────────        │ │                 │ │
│     │ 工具：画框/平移   │ │   底图 + 标注框  │ │
│     │ ─────────        │ │                 │ │
│     │ 标注列表          │ │                 │ │
│     │  #1 禮 x%,y%     │ │                 │ │
│     │  #2 ...          │ └─────────────────┘ │
│     │ ─────────        │                     │
│     │ 画框后 → 表单弹出 │                     │
└─────┴──────────────────┴─────────────────────┘
```

### 6.4 功能清单

保留旧 `AnnotationModule.tsx` 的能力（代码逻辑可大量复用，只换 OSD 坐标换算部分）：

| 能力 | 实现要点 |
|---|---|
| 画框 | 监听 `pointerdown/move/up`，记录容器归一化坐标 → 换算到图像归一化坐标 `(x, y, w, h) ∈ [0,1]` |
| 表单 | glyph（单字）/ label / note / author 四字段，用 POST `/api/relics/{id}/annotations` |
| 列表 | 左侧栏滚动卡片（小号卡片，内嵌圆角），点击定位到图上中心 |
| 选中联动 | 图上框高亮 + 列表卡片高亮双向同步 |
| 编辑/删除 | 卡片上两个小按钮；PATCH/DELETE `/api/annotations/{id}` |
| 工具切换 | "画框 / 平移" 两按钮切换容器的鼠标行为 |

### 6.5 新能力（可选）

- 键盘 `Esc` 取消画框、`Delete` 删除选中
- 标注框左上角显示数字编号（解决 glyph 为空时框看起来没标识的问题）
- 右下角缩放按钮组（Reset / Fit / 100%），体验上比纯滚轮好用
- 导出按钮：把当前文物所有标注下载为 `.json`

### 6.6 落地步骤（建议顺序）

1. 重写 `src/modules/AnnotationModule.tsx`：
   - 复制多图层模块已验证的 `LayerViewport` 平移缩放逻辑（但那个文件已删，可参考本文档 §2.3 删除列表之前版本的 `MultiLayerModule.tsx`，或从 git `HEAD^` 找回 —— 最简单是新写一个 `AnnotationViewport` 组件）
   - 保留旧的 `AnnotationRect` / `AnnotationForm` 子组件，但移除所有 OSD 引用，改用归一化 % 定位
   - 布局严格按 §4 UI 约定
2. 验证：`npx tsc --noEmit` + 手动测试画框 → 保存 → 重开能加载
3. （可选）加导出 JSON 功能
4. 更新本文档 §1 的状态为 ✅

### 6.7 注意事项

- **后端 API 完全可用**：`/api/relics/{id}/annotations` 所有方法 + `/api/annotations/{id}` PATCH/DELETE 已在生产代码里，不需要改后端
- 标注坐标是 **归一化** 存储（`bboxX` 等为 0~1 浮点），这意味着切换底图产物（原始/微痕/线图）标注框都能对齐 — 下一步实现时可考虑让用户在"当前底图"下拉里切换，观感更好
- `ImageProcessingProduct` 类型定义里的 `width` / `height` 是产物图像像素尺寸，画框坐标换算时不需要它 —— 归一化坐标天然不依赖像素尺寸
