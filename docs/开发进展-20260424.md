## 3. OCR 技术选型记录

### 3.1 为什么不用 cnocr

首次选型时用了 `cnocr 2.3.2.3`，实际安装后发现：

```
TypeError: argument should be a str or an os.PathLike object ...
File "cnstd/ppocr/rapid_detector.py": rapidocr 调用时 model_root_dir=None
```

`cnocr 2.3` 强制依赖 `rapidocr>=3.0`，但 `cnstd 1.2.7` 里的 `rapid_detector.py` 给 `RapidOCR` 传了 `model_root_dir=None`，`rapidocr 3.x` 无法处理。初始化 `CnOcr()` 就抛异常，`/api/ocr` 恒 500。

### 3.2 直接用 rapidocr

- **同样的 ONNX 引擎**：`cnocr` 底层就是 `rapidocr`，去掉包装反而更干净
- **API 极简**：`engine = RapidOCR(); result = engine(img_bgr)`；`result.txts / result.scores / result.boxes`
- **模型总计 ~15MB**：PP-OCRv4 mobile 系列，碑刻文字识别足够
- **无 `cnstd` 包装层 bug**

```python
_ocr_instance: Any = None

def _get_ocr() -> Any:
    global _ocr_instance
    if _ocr_instance is not None:
        return _ocr_instance
    from rapidocr import RapidOCR
    _ocr_instance = RapidOCR()
    return _ocr_instance
```

### 3.3 前端 OCR 交互

- 只在 **draft** 表单上出现「识别」按钮（已保存的标注不再识别）
- 点击后：`ocrLoading=true` → `ocrRegion()` 调后端 → 结果回填：
  - 单字 → 填 `glyph`
  - 多字 → 首字填 `glyph`，全文填 `label`
  - 空结果 → `ocrError = "未识别到文字"`
- `ocrLoading / ocrError` 与 `saving / error` 解耦，互不影响

## 4. 视口缩放保持

原 `AnnotationViewport` 在每次 `imgUrl` 变化时 `setImgReady(false)` + 新图 load 后无条件 `fit()`，**用户调参 / 切底图都会丢失当前缩放**。

改动：

```ts
const lastSizeRef = useRef<{ w: number; h: number } | null>(null)

const handleImgLoad = () => {
  const img = imgRef.current
  const nw = img?.naturalWidth ?? 0
  const nh = img?.naturalHeight ?? 0
  const last = lastSizeRef.current
  const sizeChanged = !last || last.w !== nw || last.h !== nh
  lastSizeRef.current = { w: nw, h: nh }
  setImgReady(true)
  if (sizeChanged) fit()   // 仅首次 / 尺寸真的变了才 fit
}
```

`ResizeObserver` 的 `fit` 也只在 `lastSizeRef.current === null` 时触发，避免用户拉伸窗口时把缩放重置。

同一文物所有产物都来自同一张 downsized 原图，长宽相同 → 切换底图 / 参数实时重渲染 / 线图参数调整 时，view 完整保留。切换到不同文物（尺寸变化）时自动 fit 回中。

## 9. 给下一个 agent 的上下文回放要点

- **OCR 引擎是 rapidocr 3.x**，不是 cnocr / paddleocr
- 标注 UI 都走 `components/annotation/` 下的公共组件 + `useAnnotationSession`，不要在业务模块里重复实现状态机
- **对比 / 标注 互斥**是写死的产品决策（分屏画框互斥），不要随便加「同时开启」的兼容逻辑
- **独立字迹标注模块可能被删**：所有依赖它的引用都应只通过 `components/annotation/` 间接依赖
- 视口（`AnnotationViewport`）的缩放保留机制依赖 `lastSizeRef`：同尺寸不 fit，尺寸变 / 首次才 fit；修改该文件时注意不要破坏该契约
